# v0.10 — LLM Backend Abstraction (Claude Code + Codex)

> v0.x 전체 기간 동안 SoloSquad는 *Claude Code Max* 단일 백엔드 가정으로 만들어졌음. 사용자 요청: **OAuth 기반 AI CLI 2종 (Claude Code, Codex)을 backend로 선택** 가능하게. 분석 결과 **구현 차단점 10건** 식별 — 실 Codex 실행은 *수 주~수 개월 refactor*. v0.10에서는 ① docs 문구 수정 (Claude Code Max 단일 의존성 해소) + ② 온보딩 wizard에 backend 선택 UX + ③ `src/llm/adapter.ts` 추상 layer 골격 + workspace.yaml `llm_backend` 필드만 도입. Codex 실 실행은 *v1.x slot* 영구 박제.

**로드맵:** v0.9.0 Model B → **v0.10 LLM Backend Abstraction** → v1.0 정식 출시

**의존:** v0.3 PM mode (Claude session 모델), v0.5 4채널 라우터, v0.8.2 dev_capability

**기준 버전:** v0.10. **구현 차단점 박제 patch + 사용자 인지 정확화** — 코드 변경은 (a) docs 다수 (b) wizard 1-step 추가 (c) adapter 골격 1 모듈 한정. 사용자 노출 표면적 *증가 1개* (wizard에 backend 선택 prompt).

**분류:** Documentation correction + UX honesty + architectural scaffolding.

## 2. Codex 백엔드 도입 시 구현 차단점 10건

본 절은 **현재 SoloSquad가 *Claude Code 전제로* 어떻게 동작하는지** 박제 + **Codex 도입 시 무엇이 깨지는지** 명시. 솔로 founder의 자기 사용 시나리오에서 *어느 backend를 쓰든 동일 UX*를 약속하려면 아래 10건 모두 해결 필요.

### 2.1 PM session 모델 부재 (가장 큰 차단)

**현재 (Claude Code)**:
```bash
claude --print --session-id <uuid> --resume \
  --output-format stream-json --input-format stream-json \
  --verbose --exclude-dynamic-system-prompt-sections
```

- v0.3+에서 PM은 *영구 session*. 사용자가 연속 메시지 보내면 *같은 session id로 resume*
- 컨텍스트가 누적됨 — 이전 대화 기억
- claude-process.ts:218-225가 `--resume` 분기

**Codex CLI**:
- session 개념 *없음*. 매 호출이 stateless
- 또는 `codex` REPL이 *terminal 안에서만* 유지 — bot subprocess가 가져가 못 함

**해결 (theoretical)**:
- Codex 어댑터에서 *외부 state* 유지 — 메시지 히스토리를 `<org>/.solosquad/sessions/<user>.codex.jsonl`에 저장
- 매 호출 직전 history 전체를 system prompt 또는 messages 배열로 prepend
- **문제**: 토큰 누적 → 비용 폭증. Claude는 캐시 hit이라 저렴, Codex는 매번 full prompt 청구

**v0.10 결정**: stub만. 실 구현은 v1.x slot에서 별도 plan (`v1.x-codex-session-shim.md`?).

### 2.2 Task tool 부재

**현재 (Claude Code)**:
- Claude 내장 `Task` tool로 specialist spawn (depth=1)
- PM이 `Task: business-strategist "..."` 호출 → 새 fresh subagent session
- pm-runner.ts:432 주석: "Task tool prompt body" 명시

**Codex**:
- 내장 spawning tool 없음
- subagent 개념 자체가 다름 (Codex는 *single chain of thought*)

**해결 (theoretical)**:
- Codex 어댑터에서 *명시 spawn* — PM의 텍스트 응답에서 `[SPAWN agent ...]` 패턴 파싱 → 별도 `codex` invocation 시작 → 결과를 PM 다음 turn에 prepend
- 또는: SoloSquad가 *외부 orchestrator* 역할 → Codex는 *agent 1개씩 호출*만, 합치는 건 SoloSquad

**v0.10 결정**: 어댑터 인터페이스에 `spawnSpecialist(prompt, cwd)` 메서드 추가. Claude 구현은 *Task tool 위임*, Codex 구현은 *throw not-implemented*.

### 2.3 Stream JSON 포맷 다름

**현재 (Claude Code)**: `--output-format stream-json --input-format stream-json` — Anthropic 표준

**Codex**: 자체 streaming protocol. `codex --json` 같은 flag 있을 수 있으나 event schema 다름

**해결**: 어댑터별 event 정규화 → 공통 `LlmStreamEvent` 타입 (start / tool_use / text_delta / tool_result / end / error)

**v0.10**: 인터페이스만 정의. Claude 구현체는 기존 stream-json 파싱 재사용. Codex 구현체는 stub.

### 2.4 SKILL.md 형식 vendor 종속

**현재**: Anthropic Agent Skills 공식 포맷 (`name`/`description` + `triggers`/`loop_mode`/`budget` 등 SoloSquad 확장)

**Codex**: skill 개념 없음. `/goal` + `AGENTS.md` (Codex 자체 표준) — *다른 영역의 추상화*

**해결 (theoretical)**:
- Codex 어댑터에서 SKILL.md → "system prompt + goal.md 합성"으로 *런타임 변환*
- 또는 vendor-neutral skill schema 정의 → 둘 다에서 변환

**v0.10**: skill format 유지 (Claude 호환 그대로). Codex 어댑터의 변환 로직은 *v1.x slot*.

### 2.5 Auth probe + login 다름

**현재 (Claude Code)**:
```bash
claude auth status --json
# {"loggedIn": true, "subscriptionType": "max", ...}
```

**Codex**:
- `codex auth status` 또는 다른 명령 (CLI 버전별 다름)
- OAuth 후 API key 발급 모델 (Claude는 *세션 자격* 모델)

**해결**: 어댑터별 `authStatus(): Promise<{ loggedIn: boolean; backend: string; details?: any }>`

**v0.10**: 인터페이스만. Claude 구현은 기존 `RealClaudeProcessFactory.authStatus()` 재사용. Codex는 stub.

### 2.6 `--append-system-prompt` 동등 X

**현재**: Claude 전용 flag. PM 시스템 프롬프트에 워크스페이스 컨텍스트 append.

**Codex**: system role messages 배열에 추가 (API 호출 시 직접) — flag 아닌 *프로그래매틱 조립*

**해결**: 어댑터별 `assembleSystemPrompt(layers: PromptLayer[])` 메서드. Claude는 flag, Codex는 messages 배열 빌드.

### 2.7 비용 추적 형식

**현재 (Claude Code)**: stream-json의 `cost` event에 `total_cost_usd`. paperclip envelope과 정합.

**Codex**: 별도 token usage report (prompt_tokens, completion_tokens). USD 환산은 SoloSquad가 직접 계산해야 함 (모델별 가격표 필요).

**해결**: 어댑터별 `parseCost(streamEvent): { usd: number }`. 공통 `<org>/memory/agent-costs.jsonl` schema 유지.

**v0.10**: 인터페이스만. Codex 가격표는 *v1.x slot* (가격 변동 frequent).

### 2.8 dev-confirm gate hook 위치

**현재**: `checkBashCommand`가 Claude bash tool call event를 가로채서 `SENSITIVE_BASH_PREFIXES` 매칭 시 30분 dev-confirm 대기.

**Codex**: bash 실행 패턴 다름. shell 도구 호출이 *별도 tool* 또는 *function call*로 표현.

**해결**: 어댑터별 hook 위치 — *tool_use event*에서 어댑터별 bash 패턴 매칭.

### 2.9 Context window 차이

| Backend | Context |
|---|---|
| Claude (claude-sonnet-4 등) | ~200k+ |
| Codex (GPT-4o) | 128k |
| Codex (GPT-5? — 2026 기준) | 미확정 |

8-layer JIT의 `max_context_tokens` (default 80,000) — Claude 200k 기준. Codex에서는 더 낮춰야 함.

**해결**: workspace.yaml의 `spawn.max_context_tokens`를 *backend별 default* 분기.

**v0.10**: workspace.yaml schema에 backend별 추천값 박제 (코드 변경 X).

### 2.10 과금 모델 차이 — *사용자 인지 부담*

| Backend | 과금 | 사용자 부담 |
|---|---|---|
| Claude Code Max | 월 정액 (Max plan ~$200/월 등) | *예측 가능* — 정액 외 추가 청구 X |
| Codex / OpenAI API | 토큰별 (input/output) | *예측 불가* — 사용량 폭증 시 거액 청구 가능 |

**해결 (인프라)**:
- workspace.yaml의 `author.daily_usd`·`weekly_usd` cap (v0.5 paperclip envelope)을 backend별 *default 강제*
- Codex backend 선택 시 *명시적 cap 입력 prompt* (예: 일 5 USD, 주 25 USD)
- doctor에 사용량 누적 알림 — 80% 도달 시 warn

**v0.10**: workspace.yaml schema에 박제. 실 cap enforcement는 기존 paperclip envelope 재사용 (vendor-agnostic).

### 3.3 `src/llm/adapter.ts` 추상 인터페이스 + Claude 구현 wrap

```typescript
// src/llm/adapter.ts (신규)
export type LlmBackend = "claude" | "codex";

export interface LlmAuthStatus {
  loggedIn: boolean;
  backend: LlmBackend;
  details?: Record<string, unknown>;
}

export interface LlmInvocation {
  prompt: string;
  cwd: string;
  sessionId?: string;
  timeoutMs?: number;
  maxBudgetUsd?: number;
}

export interface LlmAdapter {
  readonly backend: LlmBackend;
  invoke(inv: LlmInvocation): Promise<string>;
  authStatus(): Promise<LlmAuthStatus>;
  // Spawn specialist sub-agent. Claude → built-in Task tool. Codex → stub in v0.10.
  spawnSpecialist?(inv: LlmInvocation): Promise<string>;
}

export function getAdapter(backend: LlmBackend): LlmAdapter {
  if (backend === "claude") {
    return new ClaudeAdapter();
  }
  if (backend === "codex") {
    return new CodexAdapter();
  }
  throw new Error(`Unknown LLM backend: ${backend}`);
}
```

**ClaudeAdapter**: 기존 `runClaude` 로직 wrap. 새 함수가 *동일 출력 보장*.

**CodexAdapter** (stub):
```typescript
export class CodexAdapter implements LlmAdapter {
  readonly backend = "codex" as const;
  invoke(): Promise<string> {
    throw new Error(
      "Codex backend is not yet implemented (v0.10 stub). " +
      "See docs/plan/v0.10-llm-backend-abstraction.md §2 for blockers. " +
      "Set workspace.yaml.llm_backend to 'claude' to use Claude Code."
    );
  }
  authStatus(): Promise<LlmAuthStatus> {
    return Promise.resolve({ loggedIn: false, backend: "codex" });
  }
}
```

기존 `runClaude` 호출자 (claude-runner.ts, scheduler/index.ts, run-routine.ts)는 **유지** — 즉시 모든 콜사이트 교체 X. *v0.10에서는 어댑터 골격만*, 점진적 마이그레이션은 v0.10.x patches에서.

### 3.4 `workspace.yaml.llm_backend` 필드
