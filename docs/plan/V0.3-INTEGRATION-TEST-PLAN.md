## 2. PM session bootstrap (no messenger yet)

**Goal:** ensure the bot can spin up the PM and produce a stream-json reply
end-to-end **without** hitting Slack/Discord.

```bash
# Manually drive PM-runner via node REPL or a tiny script
node --input-type=module -e "$(cat <<'EOF'
import { RealClaudeProcessFactory } from "./dist/src/bot/claude-process.js";
import { PmRunner } from "./dist/src/bot/pm-runner.js";
import { SessionStore } from "./dist/src/bot/session-store.js";
import { FileEventSink, pmEventsPath } from "./dist/src/bot/events.js";
import path from "path";

const WORKSPACE = process.env.WS || process.cwd();
const ORG = process.env.ORG || "your-org-slug";
const USER = "test-user-1";

const claude = new RealClaudeProcessFactory();
const sessions = new SessionStore(WORKSPACE);
const pm = new PmRunner({
  claude,
  sessions,
  events: (org, user) => new FileEventSink(pmEventsPath(WORKSPACE, org, user)),
});

const reply = await pm.handleUserMessage({
  userId: USER,
  orgSlug: ORG,
  orgCwd: path.join(WORKSPACE, ORG),
  userText: "Hi. What specialists do you see on your roster?",
});
console.log("REPLY:", reply.text);
console.log("COST:", reply.costUsd, "DURATION:", reply.durationMs, "ms");
EOF
)"
```

**Expect:**
- Exit 0
- `REPLY:` contains a list of specialist names (desk-researcher, ui-designer,
  backend-developer, etc.) — proves `.claude/agents/` discovery worked
- `COST:` ~$0.10 first call (cache_creation), `DURATION:` 3–10s
- `<org>/.solosquad/sessions/test-user-1.json` created with a session UUID
- `<org>/.solosquad/sessions/test-user-1.events.jsonl` has `pm.message_in` +
  `pm.message_out` lines

Run the script **twice** — the second invocation should be ~5–6× cheaper
(resume cache hit, per PoC #1 measurements).

## 4. Auth-expired path

**Goal:** verify the bot surfaces a clear hint instead of failing silently
when Claude Code is logged out.

```bash
claude auth logout            # destructive — only on a test box
node /path/to/dist/bin/solosquad.js bot   # in a separate terminal
```

Send any message in `#owner-command`. The bot should reply with:

> 🔐 Claude Code is not logged in. Run `claude login` on the host running
> this bot, then try again.

After `claude login`, the next message should succeed without restarting
the bot.

## 5. Concurrent message serialization

**Goal:** confirm the per-session mutex prevents jsonl interleaving.

Send two messages in `#owner-command` within ~500ms (paste two messages
back-to-back). The bot should reply to both. Check the Claude Code session
jsonl (`~/.claude/projects/<cwd>/<sid>.jsonl`) — message sequences should be
strictly serial (no interleaved user/assistant blocks).

## 6. Long-running session resume (cache reuse)

**Goal:** confirm prompt-cache hit rate stays high on a real conversation.

Have a 5–10-turn back-and-forth in `#owner-command`. After each turn, check
`events.jsonl` for `pm.message_out.costUsd`. Costs should decrease after
the first turn and stabilize in the $0.02–$0.04 range as long as turns
are within ~4 minutes of each other (Anthropic prompt cache TTL = 5 min).

## 7. Session rotation on lost session

**Goal:** confirm "No conversation found" → rotate + retry works.

```bash
# Trigger by manually deleting the Claude Code session file
rm ~/.claude/projects/<encoded-cwd>/<session-uuid>.jsonl

# Send next message in #owner-command
```

**Expect:**
- Bot replies normally (single noticeable pause for the retry)
- `events.jsonl` has a `pm.session_lost` line with old + new session IDs
- `<org>/.solosquad/sessions/<user>.json` has a non-empty `archived` array

## 8. Rollback

**Goal:** confirm we can revert if anything breaks.

```bash
node /path/to/dist/bin/solosquad.js migrate --rollback
```

Workspace returns to `version: 1.2.4`. Run a v1.2.4 build of `solosquad bot`
to confirm pre-PM behavior still works.
