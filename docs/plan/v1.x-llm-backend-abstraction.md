# v1.x — LLM Backend Abstraction (Codex Adapter)

> v0.10.0 시도(`c02fcb5 feat(v0.10.0): LLM backend abstraction`)는 merge 후 `2f162f7`로 **revert** 됨. v0.x는 Claude Code (OAuth) 단일 백엔드로 마무리하고, 멀티 백엔드 — 특히 Codex — 는 **v1.x 슬롯으로 영구 박제**. 본 문서는 *시도 시점에 식별된 10개 구현 차단점*과 *해법 방향*을 v1.0 출시 이후 재진입할 때 참조할 수 있도록 정리한다.

**로드맵 위치:** v0.9.x 자체 사용 검증 → **v1.0 정식 출시 (Claude Code single backend)** → **v1.x LLM Backend Abstraction 본 슬롯** → v1.1 대시보드 → v1.2 지식 온톨로지

**기준 버전:** v1.x slot (구체 minor/patch는 v1.0 출시 후 ideation 재진입 시점에서 split).

**의존:** v0.3 PM session 모델, v0.5 paperclip envelope, v0.8.2 dev_capability, v1.0 (Claude Code 단일 백엔드로 API stability 시작).

**분류:** Architectural refactor — vendor-neutral LLM adapter layer.

## 0. 본 슬롯이 무엇인지 / 무엇이 아닌지

- *v1.x에서 무조건 한다*는 약속 X — v1.0 이후 사용자 / 본인이 실제로 Codex 백엔드를 필요로 할 때까지 deferred 유지

## 2. 10가지 구현 차단점 (architectural 박제)

본 절은 v0.x SoloSquad가 *Claude Code 전제로 어떻게 동작하는지* + *Codex 도입 시 무엇이 깨지는지*. 본 슬롯에서 재진입 시 각 차단점이 별도 sub-slot으로 분리될 수 있다.

### 2.1 PM session 모델 부재 *(가장 큰 차단)*

| 측면 | Claude Code | Codex |
|---|---|---|
| Session | `--session-id <uuid> --resume` 영구 session, 컨텍스트 누적, 캐시 hit | session 개념 없음. stateless 호출, 또는 REPL이 terminal에만 |
| 비용 | Max 정액 (캐시 효과로 저렴) | 매 호출 full prompt 청구 → 누적 폭증 위험 |

**해법 방향**: Codex 어댑터에서 *외부 state* (`<org>/.solosquad/sessions/<user>.codex.jsonl`) 유지 + history-prepend session 시뮬레이션. 단 토큰 누적 비용 모델 필요.

**별도 slot 후보**: `v1.x-codex-session-shim.md`.

### 2.2 Task tool 부재 (specialist spawning)

| 측면 | Claude Code | Codex |
|---|---|---|
| Spawn | 내장 `Task` tool로 specialist depth=1 spawn | 내장 spawning 없음. single chain of thought |
| PM 위임 | pm-runner.ts:432 — Claude 내장에 위임 | SoloSquad가 외부 orchestrator로 explicit spawn 필요 |

**해법 방향**: `LlmAdapter.spawnSpecialist(prompt, cwd)`. Claude는 Task 위임, Codex는 별도 `codex` invocation + 결과 prepend.

### 2.3 Stream JSON 포맷 다름

Claude: `--output-format stream-json --input-format stream-json` (Anthropic 표준). Codex: 자체 streaming, event schema 다름.

**해법 방향**: 공통 `LlmStreamEvent` 타입 (start / tool_use / text_delta / tool_result / end / error) + per-backend parser.

### 2.4 SKILL.md 형식 vendor 종속

Claude: Anthropic Agent Skills 공식 포맷 + SoloSquad 확장 (`triggers`/`loop_mode`/`budget`). Codex: skill 개념 없음, `/goal` + `AGENTS.md` (cross-tool 표준).

**해법 방향**: 런타임 변환 (SKILL.md → system prompt + goal.md 합성) 또는 vendor-neutral skill schema 재설계.

**별도 slot 후보**: `v1.x-skill-format-vendor-neutral.md`.

### 2.5 Auth probe + login 다름

Claude: `claude auth status --json` → `{loggedIn, subscriptionType, ...}`. Codex: `codex auth status` (CLI 버전별), OAuth → API key 발급 모델.

**해법 방향**: `LlmAdapter.authStatus(): Promise<{loggedIn, backend, details?}>`.

### 2.6 `--append-system-prompt` 부재

Claude: CLI flag로 워크스페이스 컨텍스트 append. Codex: system role messages 배열에 프로그래매틱 조립.

**해법 방향**: `LlmAdapter.assembleSystemPrompt(layers)` — Claude는 flag, Codex는 messages 빌드.

### 2.7 비용 추적 형식

Claude: stream-json `cost` event에 `total_cost_usd`, paperclip envelope과 정합. Codex: prompt/completion token counts 별도, USD 환산은 SoloSquad가 가격표 유지하며 직접 계산.

**해법 방향**: `LlmAdapter.parseCost(event): {usd}`. 공통 `<org>/memory/agent-costs.jsonl` schema 유지. Codex 가격표는 별도 slot (모델 가격 변동 잦음).

### 2.8 dev-confirm gate hook 위치

Claude: `checkBashCommand`가 bash tool call event 가로채서 `SENSITIVE_BASH_PREFIXES` 매칭. Codex: shell 호출이 별도 tool / function call로 표현 — 패턴 다름.

**해법 방향**: 어댑터별 tool_use event 위치 + bash 패턴 매칭.

### 2.9 Context window 차이

| Backend | Context |
|---|---|
| Claude (sonnet-4 등) | ~200k+ |
| Codex (GPT-4o) | 128k |
| Codex (GPT-5? 2026) | 미확정 |

8-layer JIT의 `spawn.max_context_tokens` (default 80,000) — Claude 200k 기준. Codex는 더 낮춰야 함.

**해법 방향**: workspace.yaml의 `spawn.max_context_tokens`를 backend별 default 분기.

### 2.10 과금 모델 차이 — *사용자 인지 부담*

| Backend | 과금 | 사용자 부담 |
|---|---|---|
| Claude Code Max | 월 정액 (~$200/월) | 예측 가능, 정액 외 추가 청구 X |
| Codex / OpenAI API | 토큰별 (input/output) | 예측 불가, 폭증 시 거액 청구 |

**해법 방향**:
- workspace.yaml `author.daily_usd` / `weekly_usd` cap을 backend별 *default 강제*
- Codex 선택 시 명시적 cap 입력 prompt 강제 (예: 일 5 USD, 주 25 USD)
- doctor에 누적 알림 — 80% 도달 시 warn

paperclip envelope 자체는 vendor-agnostic이므로 재사용 가능.

## 5. 본 슬롯 재진입 트리거

다음 조건 중 *하나라도* 만족하면 본 슬롯을 v1.x 정식 plan으로 split하고 진입:

1. **사용자(본인) workflow에서 Codex 병행 사용 시작** — dev 환경에 Codex/Cursor 병행 시 SoloSquad가 그 백엔드로도 spawn할 필요가 실제로 발생
2. **외부 사용자 요청 누적** — v1.0 출시 후 *Claude Code 외 백엔드* 지원 요구가 누적되는 시점
3. **Codex (또는 다른 OAuth CLI) feature parity 달성** — session 모델·Task tool·stream JSON 등 §2 차단점 중 다수가 vendor 측에서 해결되어 SoloSquad의 shim 부담이 크게 줄어드는 시점
4. **paperclip envelope이 multi-tenant**로 확장 — 본 슬롯의 §2.10 과금 분기가 본 envelope feature와 합쳐서 더 큰 plan으로 통합되는 시점

→ 위 trigger 발생 전에는 본 슬롯은 *deferred*. v1.0~v1.x의 다른 plan들이 우선.
