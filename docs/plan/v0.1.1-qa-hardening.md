## 3. QA Engineer 에이전트 추가

### 신규 파일
- `assets/agents/engineering/qa-engineer/SKILL.md`

### 라우팅 키워드 (5개)
`테스트`, `qa`, `품질`, `버그`, `검증`

### 워크플로우 반영
- PMF 탐색 (Type A): Development → **QA + Marketing (병렬)** → Launch
- 가설 검증 (Type B): Development → **QA**
- 기능 확장 (Type C): Development → **QA**
- 빠른 프로토타입 (Type D): Development → **QA** → Marketing

### status.yaml 변경
- `qa` stage 추가 (Phase 5, depends_on: development)
- `marketing` Phase 4 → Phase 5
- `launch` Phase 5 → Phase 6, depends_on에 `qa` 추가

### TEAM_KNOWLEDGE.md 추가 항목
- 테스트 피라미드 (Unit → Integration → E2E)
- 버그 심각도 분류 (Critical / Major / Minor / Trivial)
- Go/No-Go 기준 (Hard Gate / Soft Gate)
