> 사용자가 에이전트 역량을 자유롭게 정의하고, 자율 실행 파이프라인에 끼워넣을 수 있는 개방형 스킬 시스템.

**로드맵:** v1.1 크로스 플랫폼 → v1.2 안전/보안 → v1.3 자율 실행 → **v1.4 스킬 자유도** → v1.5 지식 온톨로지 → v1.6 웹 대시보드 → v2.0 커뮤니티 → v3.0 코워킹

## 1. 현재 문제

| 문제 | 원인 |
|------|------|
| 에이전트 23개 고정 | 사용자가 새 스킬을 추가하려면 코드 수정 필요 |
| 라우팅이 하드코딩 | `AGENT_ROUTES` 딕셔너리에 키워드 수동 등록 |

handoff_to:
  - feature-planner

## 3. 자동 라우팅 등록

현재: `AGENT_ROUTES` 하드코딩 (agent-router.ts)
개선: **파일 시스템 스캔 + 프론트매터 파싱으로 자동 등록**

```typescript
// 개선된 agent-router.ts
export async function buildRoutes(): Promise<Record<string, [string, string]>> {
  const routes: Record<string, [string, string]> = {};

  // 1. 번들 에이전트 스캔 (assets/agents/**/SKILL.md)
  // 2. 사용자 커스텀 에이전트 스캔 (~/.solo-agents/agents/**/SKILL.md)
  // 3. 프로젝트 로컬 에이전트 스캔 (./.agents/**/SKILL.md)

  // 프론트매터의 triggers → 라우팅 키워드로 자동 등록
  // 우선순위: 프로젝트 로컬 > 사용자 커스텀 > 번들
  return routes;
}
```

## 4. 스킬 검색 경로 (3-tier)

```
1. ./.agents/{team}/{agent}/SKILL.md          → 프로젝트 로컬 (최우선)
2. ~/.solo-agents/agents/{team}/{agent}/SKILL.md  → 사용자 글로벌
3. assets/agents/{team}/{agent}/SKILL.md      → 번들 기본
```

- 같은 이름이면 상위 계층이 override
- `solo-agents agent add` CLI로 커스텀 에이전트 스캐폴딩

---

## 5. 프로세스 통합

program.md(v1.2)의 에이전트 파이프라인에 커스텀 에이전트를 자유롭게 배치:

```markdown
## 에이전트 파이프라인
1. desk-researcher: 시장 데이터 수집          # 번들 에이전트
2. custom-analyst: 도메인 특화 분석            # 사용자 커스텀 에이전트
3. data-analyst: 종합 인사이트 도출            # 번들 에이전트
4. my-report-writer: 최종 보고서 작성          # 프로젝트 로컬 에이전트
```

---

## 6. CLI

```bash
# 커스텀 에이전트 생성 (대화형 위저드)
solo-agents agent add --name "custom-analyst" --team strategy

# 등록된 에이전트 목록 (번들 + 커스텀 + 로컬)
solo-agents agent list

# 에이전트 상세 정보
solo-agents agent info custom-analyst

# 에이전트 검증 (프론트매터 + 필수 섹션)
solo-agents agent validate ./.agents/strategy/custom-analyst/SKILL.md
```
