## 1. 현재 문제

| 문제 | 위치 | 설명 |
|------|------|------|
| `which` 명령 Unix 전용 | `init.ts`, `doctor.ts` | Windows에는 `which`가 없음. `where`로 분기하지만 불안정 |
| 경로 구분자 하드코딩 | 곳곳 | `/` 직접 사용. Windows는 `\` |
| `sudo` 필요 여부 | `update.ts` | Linux/macOS에서 글로벌 npm 설치 시 `sudo` 필요할 수 있음 |
| 셸 차이 | CLI 전반 | bash/zsh vs PowerShell/cmd — 환경변수, 경로 확장 등 |
| Docker Desktop 필수 | `init.ts` | Windows/macOS에서 Docker Desktop 라이선스 이슈 |
| 줄바꿈 차이 | Git + 파일 파싱 | CRLF(Windows) vs LF(Unix) — JSONL/TSV/MD 파싱 영향 |
| HOME 경로 | `paths.ts` | `~` 확장이 Windows에서 다르게 동작 |

## 2. 설치 방법 (3개 OS)

### npm 글로벌 설치 (공통)

```bash
# 모든 OS 공통
npm install -g solosquad
solosquad init
```

### OS별 사전 준비

**macOS:**
```bash
# Homebrew로 의존성 설치
brew install node git
brew install --cask docker   # 선택

# Claude Code CLI
npm install -g @anthropic-ai/claude-code
```

**Windows:**
```powershell
# winget으로 의존성 설치
winget install OpenJS.NodeJS.LTS
winget install Git.Git
winget install Docker.DockerDesktop   # 선택

# Claude Code CLI
npm install -g @anthropic-ai/claude-code

# 권장: Windows Terminal + PowerShell 7
winget install Microsoft.WindowsTerminal
winget install Microsoft.PowerShell
```

**Linux (Ubuntu/Debian):**
```bash
# apt로 의존성 설치
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs git
sudo apt install -y docker.io   # 선택

# Claude Code CLI
npm install -g @anthropic-ai/claude-code

# npm 글로벌 권한 문제 해결 (sudo 없이 설치하려면)
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

## 3. 코드 수정 사항

### 3.1 명령어 존재 확인 통합

현재 `which`/`where` 분기를 Node.js 네이티브 방식으로 교체:

```typescript
// src/util/platform.ts (신규)
import { execSync } from "child_process";
import os from "os";
import path from "path";

export const IS_WINDOWS = os.platform() === "win32";
export const IS_MACOS = os.platform() === "darwin";
export const IS_LINUX = os.platform() === "linux";

/** 명령어 존재 확인 — OS 무관 */
export function commandExists(cmd: string): boolean {
  try {
    const check = IS_WINDOWS ? `where ${cmd}` : `command -v ${cmd}`;
    execSync(check, { stdio: "ignore" });
    return true;
  } catch {
    return false;
  }
}

/** 홈 디렉토리 — OS 무관 */
export function homeDir(): string {
  return os.homedir(); // Windows: C:\Users\xxx, Unix: /home/xxx
}

/** 글로벌 설정 디렉토리 */
export function globalConfigDir(): string {
  if (IS_WINDOWS) {
    return path.join(process.env.APPDATA || homeDir(), "solosquad");
  }
  return path.join(homeDir(), ".solosquad");
}

/** npm 글로벌 설치 명령 (sudo 자동 판단) */
export function npmGlobalInstallCmd(pkg: string): string {
  if (IS_WINDOWS) return `npm install -g ${pkg}`;
  // Unix: root면 sudo 불필요, 아니면 sudo 사용
  const needsSudo = process.getuid?.() !== 0;
  return needsSudo ? `sudo npm install -g ${pkg}` : `npm install -g ${pkg}`;
}
```

### 3.3 줄바꿈 정규화

```typescript
// src/util/platform.ts 추가

/** 파일 내용의 줄바꿈을 LF로 정규화 */
export function normalizeLine(content: string): string {
  return content.replace(/\r\n/g, "\n");
}

/** JSONL 파싱 시 CRLF 안전 처리 */
export function parseJsonl<T>(content: string): T[] {
  return normalizeLine(content)
    .split("\n")
    .filter((line) => line.trim())
    .map((line) => JSON.parse(line));
}

/** TSV 파싱 시 CRLF 안전 처리 */
export function parseTsv(content: string): Record<string, string>[] {
  const lines = normalizeLine(content).split("\n").filter((l) => l.trim());
  if (lines.length < 2) return [];
  const headers = lines[0].split("\t");
  return lines.slice(1).map((line) => {
    const values = line.split("\t");
    return Object.fromEntries(headers.map((h, i) => [h, values[i] || ""]));
  });
}
```

```typescript
console.log(chalk.dim(`Shell: ${process.env.SHELL || process.env.ComSpec || "unknown"}`));

// Docker 체크 개선 — 선택 사항으로 변경

  console.log(` ${chalk.yellow("△")} Docker (optional) — needed for isolated execution`);

  check("Docker", true);
}

// Windows 특화 체크

  check("PowerShell 7+", commandExists("pwsh"), "winget install Microsoft.PowerShell");
  check("Windows Terminal", commandExists("wt"), "winget install Microsoft.WindowsTerminal");
}
```

## 4. CI/CD 매트릭스 테스트

GitHub Actions로 3개 OS에서 자동 테스트:

```yaml
# .github/workflows/ci.yml
name: CI
on: [push, pull_request]

jobs:
  test:
    strategy:
      matrix:
        os: [ubuntu-latest, macos-latest, windows-latest]
        node: [18, 20, 22]
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node }}
      - run: npm ci
      - run: npm run build
      - run: npm test
      - name: Smoke test (install + doctor)
        run: |
          npm install -g .
          solosquad doctor --ci
```
