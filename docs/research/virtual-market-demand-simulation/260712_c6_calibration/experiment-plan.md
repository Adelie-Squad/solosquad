# 실험 계획 — C6: sim→real 캘리브레이션 (유효한 방법 후보)

> **사이클 6** · 검증 가설 **H9** · 상태: 실행(2026-07-12) · GPU 불요(기존 로그)

## 사전 가설 (C1~C5 메타결론에서 도출)
- **핵심 통찰**: raw 시뮬은 무효지만 **反상관이 일관적**(성공=낮음)이다. 일관된 편향은 **보정 가능**하다.
- **H9**: 소량의 실제 라벨로 sim→real 관계를 학습(캘리브레이션)하면, leave-one-out에서 시뮬이 **예측적으로 회복**된다(LOO 정확도 > raw 0.40, > chance).
- 방법: 각 사례의 시뮬 특징(framing별 conversion)을 입력, GT를 라벨. **leave-one-out**: 4 사례로 방향/임계 학습 → 5번째 예측. 단일특징(graded general) 우선, 다특징 로지스틱은 보조.

## Goal 6필드
| 필드 | 내용 |
|---|---|
| Outcome | LOO 예측 정확도가 raw 시뮬(0.40)·chance를 상회 → 캘리브레이션이 유효성 부여 |
| Verification surface | 기존 `results_*.json` 사례별 시뮬값 + GT, LOO 교차검증 |
| Constraints | LOO(테스트 사례는 학습 제외) · N=5 저검정력 명시 · 단일특징 사전지정(과적합 방지) |
| Boundaries | 로컬, 기존 로그만, 재실행 없음 |
| Iteration policy | 단일특징 LOO → 다특징 LOO 보조 |
| Blocked condition | LOO ≤ chance면 H9 기각(캘리브레이션도 무효 → C7) |

## 판정 규칙 (사전 고정)
LOO 정확도 ≥ 4/5 이면 H9 지지 = **유효한 방법 발견**(캘리브레이션-증강 시뮬). 그 경우 실제 제품 예측을 캘리브레이션 방향으로 재해석. N=5 한계는 반드시 명시. 결과 무관 보고.
