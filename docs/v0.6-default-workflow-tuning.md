## 3. 의도적으로 미루는 것

- **신규 templating 엔진** — 4종이 너무 다양해지면 별도 엔진 도입 필요할 수 있으나, v0.6에서는 *기존 yaml 포맷 유지* + 데이터 튜닝만
- **GUI 편집** — v1.1 대시보드 상호작용에서 read-only 시각화는 가능. 디폴트 워크플로우 편집은 메신저 author 루프(v0.5)로 충분
- **온톨로지 매핑** — v1.2과 자연스럽게 연계되도록 의미 태깅 *결합점만* 정의, 실제 매핑은 v1.2
