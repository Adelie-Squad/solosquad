# v1.1.2 — npm 패키지 퍼블리시

> solosquad를 npm registry에 퍼블리시하여 `npm install -g solosquad`로 전 세계 누구나 설치 가능하게 만든다.

**기준 버전:** v1.1.1 (QA 강화)

---

## 1. 목표

```
npm install -g solosquad
solosquad init     # 즉시 사용 가능
```

npm 퍼블리시 = 패키지를 registry.npmjs.org에 업로드하는 것. 이후 누구나 `npm install`로 다운로드 가능.

---

## 2. 현재 상태 점검

### 이미 준비된 것

| 항목 | 상태 | 위치 |
|------|------|------|
| package.json `name` | `solosquad` | `package.json` |
| package.json `version` | `0.2.0` | `package.json` |
| `bin` 필드 | `solosquad` → `./dist/bin/solosquad.js` | `package.json` |
| `files` 필드 | `["dist/", "assets/"]` | `package.json` |
| `engines` | `node >= 18` | `package.json` |
| `prepublishOnly` | `npm run build` | `package.json` |
| `license` | MIT | `package.json` |
| `keywords` | ai, assistant, solo-founder 등 7개 | `package.json` |

### 추가 필요한 것

| 항목 | 설명 | 우선순위 |
|------|------|---------|
| npm 계정 | npmjs.com 계정 생성 + 로그인 | 필수 |
| 패키지명 확인 | `solosquad` 사용 가능 여부 확인 | 필수 |
| `repository` 필드 | GitHub 리포 URL | 권장 |
| `homepage` 필드 | 프로젝트 홈페이지/README | 권장 |
| `bugs` 필드 | 이슈 트래커 URL | 권장 |
| `author` 필드 | 작성자 정보 | 권장 |
| `.npmignore` | 퍼블리시에서 제외할 파일 목록 | 필수 |
| `CHANGELOG.md` | 버전별 변경사항 | 권장 |
| README.md 정비 | npm 페이지에 표시됨 | 권장 |

## 3. 구현 계획

### Phase 1: package.json 보완

```jsonc
// 추가할 필드
{
  "version": "1.1.2",
  "author": "YOUR_NAME <your@email.com>",
  "repository": {
    "type": "git",
    "url": "https://github.com/YOUR_USERNAME/solosquad.git"
  },
  "homepage": "https://github.com/YOUR_USERNAME/solosquad#readme",
  "bugs": {
    "url": "https://github.com/YOUR_USERNAME/solosquad/issues"
  }
}
```

### Phase 2: .npmignore 생성

npm publish 시 불필요한 파일 제외. `files` 필드가 이미 allowlist 역할을 하지만, 혹시 모를 누출 방지.

```
# 소스 (dist만 배포)
src/
bin/
tsconfig.json

# 개발
.claude/
.env
.env.example
scripts/

# 문서 (README.md는 자동 포함)
docs/
CONTRIBUTING.md

# Git/CI
.github/
.gitignore

# Docker (npm 패키지에 불필요)
Dockerfile
docker-compose.yml

# 프로젝트 데이터
projects/
memory/
repos/

# 레거시
requirements.txt
core/
routines/
templates/
agents/
orchestrator/
```

### Phase 3: 퍼블리시 전 검증

```bash
# 1. 빌드
npm run build

# 2. 패키지 내용물 확인 (실제 퍼블리시 안 함)
npm pack --dry-run
# → dist/, assets/, package.json, README.md, LICENSE만 포함되는지 확인

# 3. 패키지 크기 확인
npm pack
ls -lh solosquad-*.tgz
# 목표: 5MB 이하

# 4. 로컬 테스트 설치
npm install -g ./solosquad-*.tgz
solosquad --help
solosquad doctor

# 5. 테스트 후 제거
npm uninstall -g solosquad
```

### Phase 4: npm 퍼블리시

```bash
# 1. npm 로그인 (최초 1회)
npm login
# Username, Password, Email, OTP(2FA) 입력

# 2. 패키지명 가용성 확인
npm view solosquad
# 404 → 사용 가능

# 3. 퍼블리시
npm publish
# prepublishOnly → npm run build 자동 실행
# → https://www.npmjs.com/package/solosquad

# 4. 확인
npm info solosquad
```

### Phase 5: 퍼블리시 후 검증

```bash
# 다른 머신 또는 클린 환경에서
npm install -g solosquad
solosquad --help
solosquad doctor
solosquad init
```

## 4. 버전 관리 전략

### Semantic Versioning (semver)

```
MAJOR.MINOR.PATCH
  1  .  1  .  2
```

| 변경 유형 | 버전 업 | 예시 |
|----------|---------|------|
| 버그 수정 | PATCH (1.1.2 → 1.1.3) | 키워드 라우팅 수정 |
| 기능 추가 (하위 호환) | MINOR (1.1.x → 1.2.0) | 자율 실행 엔진 |
| 호환성 깨짐 | MAJOR (1.x → 2.0.0) | 설정 포맷 변경 |

### 릴리스 워크플로우

```bash
# 1. 버전 범프
npm version patch   # 1.1.2 → 1.1.3
npm version minor   # 1.1.x → 1.2.0

# 2. 자동으로 git tag 생성 + commit
# → v1.1.3 태그

# 3. 퍼블리시
npm publish

# 4. (선택) GitHub Release 생성
gh release create v1.1.3 --notes "버그 수정: ..."
```

### 배포 채널 (향후)

```bash
# 안정 버전 (기본)
npm install -g solosquad

# 베타 버전
npm publish --tag beta
npm install -g solosquad@beta

# 특정 버전
npm install -g solosquad@1.2.0
```

## 5. self-update 연동

이미 구현된 `solosquad update` 명령이 npm registry를 확인하도록 동작:

```
solosquad update
→ npm view solosquad version  (최신 버전 확인)
→ 현재 버전과 비교
→ npm install -g solosquad@latest  (업데이트 실행)
```

## 6. 보안 체크리스트

| 항목 | 확인 |
|------|------|
| `.env` 파일이 패키지에 포함되지 않는가 | `files` 필드 + `.npmignore` |
| API 키가 하드코딩되어 있지 않은가 | 코드 검색 |
| `postinstall` 스크립트에 위험한 명령이 없는가 | package.json 확인 |
| 2FA 활성화 | npm 계정 설정 |
| `npm audit` 취약점 0건 | 퍼블리시 전 실행 |

## 9. 퍼블리시 순서 요약

```
1. package.json 보완 (author, repository, homepage, bugs)
2. .npmignore 생성
3. npm run build
4. npm pack --dry-run → 내용물 확인
5. npm pack → 로컬 테스트 설치 → solosquad doctor
6. npm login (최초 1회)
7. npm view solosquad (이름 확인)
8. npm publish
9. 클린 환경에서 npm install -g solosquad → 검증
10. docs 업데이트 (README, cloud-deployment, setup-guide)
```
