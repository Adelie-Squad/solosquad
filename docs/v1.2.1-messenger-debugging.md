# v1.2.1 — 메신저 연결 실패 디버깅 (Slack 트리거)

> 다른 데스크탑에서 `solosquad init`으로 메신저 설정을 완료해도 `solosquad bot` 실행 시 연결되지 않는 이슈를 추적/수정한다. Slack 사용자 리포트에서 출발했으나 근본 원인은 Discord·Telegram에도 동일하게 영향.

**기준 버전:** v1.1.2 (npm 퍼블리시)
**영향 범위:** Discord / Slack / Telegram 3개 플랫폼 (Slack에서 가장 혼란스럽게 발현)
**심각도:** Critical — 초기 사용자가 첫 실행에서 반드시 부딪히는 경로

---

## 1. 재현 경로 (사용자 리포트)

```
1. 타 데스크탑에서 npm install -g solosquad
2. solosquad init → Slack 선택 → xoxb-..., xapp-... 입력
3. "✓ .env saved", "Setup Complete!" 메시지 확인
4. Slack 앱에 봇 생성 후 워크스페이스 설치 완료
5. solosquad bot 실행 → Slack 연결 안 됨 (실제로는 Discord 에러 메시지 출력)
```

---

## 2. 근본 원인 (Root Cause)

### 2.1 치명적: `.env`가 `process.env`로 로드되지 않음

| 항목 | 상태 |
|------|------|
| `package.json` dependencies | `dotenv ^16.4.0` 선언됨 |
| `dotenv.config()` 호출 | **없음** (`src/`+`bin/` 전체 grep 0건) |
| `import "dotenv/config"` | **없음** |
| 영향 | `.env`에 저장된 모든 값(토큰, `MESSENGER`, `REPOS_BASE_PATH`, `SLACK_COMMAND_CHANNEL` 등)이 런타임에서 보이지 않음 |

**호출 체인:**

```
bin/solosquad.ts                ← dotenv 로드 없음 (3줄짜리)
  → src/cli/index.ts
  → src/bot/index.ts::startBot()
  → src/messenger/index.ts::createAdapters()
      → process.env.MESSENGER  ← undefined, "discord"로 폴백
  → DiscordAdapter.startBot()
      → process.env.DISCORD_TOKEN  ← undefined
      → "DISCORD_TOKEN is not set" 출력 후 process.exit(1)
```

**제약:** `slack-adapter.ts:14`의 `COMMAND_CHANNEL = process.env.SLACK_COMMAND_CHANNEL || "owner-command"`은 **모듈 로드 시점**에 평가된다. 즉 `dotenv`가 어댑터 모듈보다 **먼저** 로드되어야 한다 → Phase 1-A는 반드시 **CLI 엔트리 최상단**의 side-effect import여야 한다.

### 2.2 3개 플랫폼 전수 증상 (동일 원인)

| 플랫폼 | `MESSENGER` 미로드 시 | `MESSENGER=X` 쉘 export 시 |
|---|---|---|
| Discord | Discord 폴백 → `DISCORD_TOKEN` undefined → `"DISCORD_TOKEN is not set"` | 동일 |
| Slack | Discord 폴백 → Slack 어댑터 **시작조차 안 됨**. 사용자는 Slack 설정을 마쳤는데 Discord 에러를 보게 되어 **가장 큰 혼란** | `"SLACK_BOT_TOKEN and SLACK_APP_TOKEN are required"` |
| Telegram | Discord 폴백 → Telegram 어댑터 **시작조차 안 됨** | `"TELEGRAM_BOT_TOKEN and TELEGRAM_CHAT_ID are required"` |

**체감 핵심:** Slack/Telegram 사용자는 `.env`에 자기 플랫폼 토큰을 넣었는데도 **Discord 토큰이 없다는 에러**를 본다. Phase 2-B(MESSENGER 출처 표시)가 이 혼동의 직접 해법.

### 2.3 `doctor`의 버그 은폐

`src/cli/doctor.ts`는 `loadEnv()`(= `.env` 파일 직접 파싱)만 참조하고 `process.env`는 보지 않는다. 결과:

- `.env`에 토큰이 있으면 `doctor`는 `"✓ All checks passed"` 출력
- 그러나 `solosquad bot`은 `process.env`를 보므로 즉시 실패
- 사용자는 자가진단으로 원인에 도달할 수 없음

## 3. 부차적 가능 원인 (Phase 1 수정 후 2차 조사)

Root cause 해결 후에도 연결되지 않을 때 플랫폼별로 점검.

### 3.1 Slack 앱 설정

| 항목 | 확인 방법 | 필수 여부 |
|------|----------|----------|
| Socket Mode 활성화 | api.slack.com/apps → Socket Mode → Enable | 필수 (`socketMode: true`) |
| App-Level Token scope | `connections:write` | 필수 |
| Bot Token scopes | `channels:read`, `channels:manage`, `chat:write`, `groups:read`, `app_mentions:read`, `channels:history` | 필수 |
| Event Subscriptions | `message.channels` | 필수 |
| 워크스페이스 재설치 | scope 변경 후 Reinstall App | 조건부 |
| `#owner-command` 채널 봇 초대 | 채널에서 `/invite @bot` | 필수 |

`slack-adapter.ts:71`은 `channelName !== COMMAND_CHANNEL`이면 **조용히 무시** — "연결 안 됨"으로 오인 가능.

### 3.2 Discord 앱 설정

| 항목 | 확인 방법 | 필수 여부 |
|------|----------|----------|
| Privileged Intent: Message Content | Developer Portal → Bot → Privileged Gateway Intents → MESSAGE CONTENT 활성화 | 필수 (`GatewayIntentBits.MessageContent`) |
| Privileged Intent: Server Members | 일부 기능 | 선택 |
| OAuth2 scopes | `bot`, `applications.commands` | 필수 |
| Bot permissions | View Channels, Send Messages, Read Message History | 필수 |
| 서버 초대 | OAuth2 URL로 서버 초대 후 `owner-command` 채널 생성 확인 | 필수 (자동 생성) |
| 서버명 ↔ product 매핑 | `discord-adapter.ts:200` — guild.name이 product.name 또는 slug를 포함해야 자동 바인딩 | 필수 |

### 3.3 Telegram 봇 설정

| 항목 | 확인 방법 | 필수 여부 |
|------|----------|----------|
| `TELEGRAM_BOT_TOKEN` | `@BotFather` → `/newbot` 결과 | 필수 |
| `TELEGRAM_CHAT_ID` | 봇과 첫 메시지 후 `https://api.telegram.org/bot<TOKEN>/getUpdates`에서 `chat.id` 확인 | 필수 |
| chat_id 포맷 | 개인/그룹 DM은 숫자(음수 가능), 채널은 `@channelname` | 필수 |
| `BotFather` Privacy Mode | 그룹 내 모든 메시지 수신하려면 Disabled | 조건부 |

### 3.4 토큰 포맷 오류 (공통)

| 항목 | 정상 포맷 | 흔한 실수 |
|------|----------|----------|
| Slack Bot Token | `xoxb-<n>-<n>-<str>` | `xoxp-`(User Token) 입력 |
| Slack App Token | `xapp-1-<str>-<n>-<str>` | Signing Secret 입력, 만료 |
| Discord Token | `<base64>.<base64>.<base64>` | Client Secret 입력 |
| Telegram Token | `<n>:<str>` | BotFather 응답 전체 복사 |
| `.env` 공백/따옴표 | `KEY=value` | 따옴표 포함 시 `saveEnv`가 그대로 보존 |

### 3.5 네트워크

- Slack: Socket Mode는 `wss://wss-primary.slack.com` 아웃바운드 WebSocket 필수
- Discord: `wss://gateway.discord.gg`
- Telegram: `api.telegram.org` HTTPS long-polling

회사 방화벽/VPN에서 차단될 수 있음.

## 4. 수정 계획

### Phase 1 — Root cause 수정 (필수)

| 단계 | 변경 | 파일 |
|------|------|------|
| 1-A | `bin/solosquad.ts` 최상단에 `import "dotenv/config"` 추가. side-effect import이므로 다른 모든 import보다 **위에** 배치해 `slack-adapter.ts:14` 같은 모듈 상수가 `process.env` 값을 읽기 전 환경이 채워지도록 함 | `bin/solosquad.ts` |
| 1-B | ~~동일 프로세스 내 `saveEnv` 후 `process.env` 즉시 반영~~ — `init`과 `bot`은 서로 다른 프로세스로 실행되므로 1-A만으로 충분. **제거** | — |
| 1-C | `doctor`를 `process.env` 기준으로 재작성. `.env` 파일 존재/형식 검사는 남기되, 토큰 검증은 `process.env`로. `.env`와 `process.env` 값이 불일치하면 경고 | `src/cli/doctor.ts` |

### Phase 2 — 사용자 진단 가능성 향상 (3 플랫폼 공통)

| 단계 | 변경 | 파일 |
|------|------|------|
| 2-A | 어댑터가 `process.exit(1)` 전에 **플랫폼별 원인**과 **다음 조치**를 다중 라인으로 출력. Slack: 토큰 포맷/Socket Mode/scopes. Discord: 토큰 포맷/MessageContent intent. Telegram: 토큰/chat_id 수집법 | `src/messenger/{slack,discord,telegram}-adapter.ts` |
| 2-B | `bot` 시작 시 `MESSENGER` 값과 **출처**를 출력 (예: `[Bot] MESSENGER=slack (from .env)`). 출처 판정은 `.env`에서 읽은 값과 `process.env` 비교 | `src/bot/index.ts` |
| 2-C | Slack 어댑터에 `app.error()` 핸들러 추가하여 Bolt 내부 에러(스코프 부족, Socket 연결 실패)를 콘솔에 노출 | `src/messenger/slack-adapter.ts` |
| 2-D | `doctor --messenger-check` 옵션 추가: 실제 API로 토큰 검증 — Slack `auth.test`, Discord `GET /users/@me`, Telegram `getMe` | `src/cli/doctor.ts` |

### Phase 3 — 문서/회귀 방지

| 단계 | 변경 | 파일 |
|------|------|------|
| 3-A | `setup-guide.md`에 3 플랫폼 각각 체크리스트 추가 (§3.1–§3.3) | `docs/setup-guide.md` |
| 3-B | README의 Discord/Slack/Telegram 섹션에 intent/scopes/chat_id 획득법 명시 | `README.md` |
| 3-C | `test/env-load.test.ts` — 임시 `.env` 생성 → Node CLI로 `bin/solosquad.ts doctor` 실행 → `process.env`에 값이 채워지고 doctor가 감지하는지 회귀 테스트 | `test/env-load.test.ts` (신규) |

## 5. 검증 체크리스트

Phase 1 수정 후 다음이 모두 통과해야 한다.

- [ ] 새 셸에서 `export MESSENGER=` 하지 않고 `solosquad bot` 실행 → `.env`의 `MESSENGER=discord|slack|telegram` 값이 로드되어 해당 어댑터가 시작됨
- [ ] Discord 경로: `[Discord Bot] Logged in: <tag>` 로그 확인
- [ ] Slack 경로: `[Slack Bot] Starting Socket Mode...` 로그 확인
- [ ] Telegram 경로: `[Telegram Bot] Polling started...` 로그 확인
- [ ] `solosquad doctor`가 `process.env` 기준으로 검사하고, `.env`와 불일치 시 경고
- [ ] 잘못된 토큰(예: Slack에 `xoxp-`) 입력 시 Phase 2-A 안내 메시지 출력
- [ ] Slack Socket Mode off 상태에서 실행 시 Bolt 에러가 `app.error()`로 콘솔에 노출
- [ ] `solosquad doctor --messenger-check` 실행 시 3 플랫폼 모두 실제 API 검증

## 7. 릴리스 노트 초안

```
v1.1.3 (hotfix) — 2026-04-xx
- Fix: .env values were never loaded into process.env, causing all messenger
  adapters (Discord/Slack/Telegram) to ignore configured tokens after
  `solosquad init`. Now `dotenv/config` is loaded at CLI entry.
- This resolves "messenger setup completed but bot doesn't connect" reports
  across all three platforms. Slack users were most affected because the
  Discord-fallback path surfaced a Discord-token error despite Slack setup.
```

```
v1.2.1 — 2026-04-xx
- Rework `solosquad doctor` to check process.env (not .env file parse), and
  warn on .env vs process.env divergence.
- Add platform-specific error guidance on adapter startup failure
  (Slack token/Socket Mode, Discord MessageContent intent, Telegram chat_id).
- Log MESSENGER value and its resolution source on bot startup.
- Add Slack Bolt `app.error()` handler to surface scope/socket errors.
- Add `solosquad doctor --messenger-check` to validate tokens against live
  APIs (Slack auth.test, Discord users/@me, Telegram getMe).
- Docs: setup-guide and README gained per-platform checklists.
- Tests: regression test for .env → process.env loading.
```
