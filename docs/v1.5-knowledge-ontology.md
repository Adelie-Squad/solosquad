## 1. 현재 문제

| 문제 | 원인 |
|------|------|
| TEAM_KNOWLEDGE.md가 수동 작성 | 프로젝트 실행 경험이 축적되지 않음 |
| 에이전트 간 지식 단절 | A 에이전트가 학습한 것을 B가 모름 |
| 프로젝트 간 지식 이전 없음 | 프로젝트 A의 교훈이 프로젝트 B에 전달 안됨 |
| 구조화 안됨 | 자연어 덩어리라 검색/참조 불가 |

## 2. 온톨로지 구조

3계층 지식 체계:

```
Knowledge Ontology
├── Team Knowledge (팀 단위)
│   ├── frameworks/        → 팀 공유 프레임워크, 방법론
│   ├── conventions/       → 팀 컨벤션, 규칙
│   └── glossary/          → 팀 용어집
│
├── Agent Knowledge (에이전트 단위)
│   ├── patterns/          → 반복 사용 패턴 (성공한 접근법)
│   ├── anti-patterns/     → 실패한 접근법 (다시 하지 말 것)
│   └── heuristics/        → 판단 기준 (언제 무엇을 선택할지)
│
└── Project Knowledge (프로젝트 단위, 자동 축적)
    ├── decisions/         → 주요 결정과 근거
    ├── experiments/       → 실험 결과 (results.tsv 요약)
    └── learnings/         → 교훈 (keep/discard에서 추출)
```

## 3. 지식 스키마

각 지식 항목은 YAML 프론트매터 + 본문:

```markdown
---
type: pattern                    # pattern | anti-pattern | heuristic | decision | learning
scope: agent                    # team | agent | project
team: strategy
agent: data-analyst              # agent scope일 때만
tags: [funnel-analysis, retention, cohort]
confidence: 0.85                 # 0-1, 실험 횟수/성공률 기반
created: 2026-03-19
updated: 2026-03-19
source: project/my-saas/cycle-15 # 어디서 학습했는지
---

# 퍼널 병목은 Activation → Retention 구간에서 가장 자주 발생

## 패턴
초기 스타트업 제품에서 퍼널 분석 시, 가장 큰 드롭오프는 Acquisition → Activation이 아니라
Activation → Retention 구간에서 발생하는 경향이 있다.

## 근거
- project/my-saas: 3회 분석 중 3회 확인 (cycle 5, 12, 15)
- project/rental-app: 2회 확인 (cycle 3, 7)

## 적용 방법
퍼널 분석 시 Activation → Retention 구간을 먼저 확인하고,
Aha Moment 도달 후 이탈하는 사용자 세그먼트를 분리 분석한다.
```

## 4. 지식 자동 축적

자율 실행 엔진(v1.2)의 keep/discard 결과에서 지식을 자동 추출:

```
사이클 완료
    ↓
keep → patterns/ 에 성공 패턴 기록 (또는 기존 패턴 confidence 상승)
discard → anti-patterns/ 에 실패 사례 기록 (또는 기존 anti-pattern confidence 상승)
    ↓
N회 이상 반복 확인된 패턴 → TEAM_KNOWLEDGE.md에 자동 승격
```

---

## 5. 에이전트 간 지식 참조

에이전트 실행 시 관련 지식을 자동 주입:

```
에이전트 실행 요청
    ↓
1. SKILL.md 로드 (역량 정의)
2. TEAM_KNOWLEDGE.md 로드 (팀 공유 지식)
3. 관련 지식 검색 (tags 기반 매칭)
   → agent knowledge: 이 에이전트의 patterns/heuristics
   → project knowledge: 현재 프로젝트의 decisions/learnings
4. 컨텍스트에 주입
```

## 7. CLI

```bash
# 지식 검색
solo-agents knowledge search "퍼널 분석"

# 지식 목록
solo-agents knowledge list --scope agent --agent data-analyst

# 지식 통계 (패턴 수, confidence 분포)
solo-agents knowledge stats

# TEAM_KNOWLEDGE.md 자동 갱신 (축적된 지식 기반)
solo-agents knowledge sync
```
