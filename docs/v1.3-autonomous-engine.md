# v1.3 — 밤새 자율 작업 완료 엔진

> "밤에 지시하고 아침에 결과를 받는다." Autoresearch 패턴을 차용한 자율 실행 루프.

**로드맵:** v1.1 크로스 플랫폼 → v1.2 안전/보안 → **v1.3 자율 실행** → v1.4 스킬 자유도 → v1.5 지식 온톨로지 → v1.6 웹 대시보드 → v2.0 커뮤니티 → v3.0 코워킹

**의존:** v1.1 (크로스 플랫폼), v1.2 (안전/보안 — 밤새 자율 실행 전 보안 기반 필수)

---

## 핵심 질문 & 성공 기준

| 핵심 질문 | 성공 기준 |
|----------|----------|
| 사람 개입 없이 지시→실행→결과물 생성이 되는가? | `solo-agents run` 1회 실행으로 8시간 자율 작업 + 아침 보고서 |

---

## 1. 현재 문제

| 문제 | 원인 |
|------|------|
| 에이전트가 1회성 | 매번 사람이 다음 단계를 지시해야 함 |
| 루틴이 보고만 함 | 인사이트 → 행동 연결 없음 |
| 실험 추적 없음 | 뭘 시도했고 뭐가 실패했는지 모름 |
| 밤새 돌릴 수 없음 | 에이전트 체이닝, 자동 판단, 시간 관리 부재 |

---

## 2. Autoresearch 패턴 차용

Karpathy의 [autoresearch](https://github.com/karpathy/autoresearch) 핵심 구조:

```
program.md (사람: 방향 설정) → train.py (AI: 실행) → 메트릭 측정 → keep/discard → 반복
```

| 원칙 | 적용 |
|------|------|
| `program.md` = 의도 전달 | 프로젝트별 `program.md`로 실행 방향 정의 |
| 단일 메트릭 판단 | keep/discard 자동 판단 기준 명시 |
| Git = 실험 메모리 | 성공만 커밋, 실패는 revert + 로그 |
| 자율 반복 루프 | 시간 예산 내 무한 반복 |

---

## 3. 자율 실행 아키텍처

```
사용자 지시 (program.md)
    ↓
┌─────────────────────────────────┐
│  Execution Engine                │
│                                  │
│  1. program.md 파싱              │
│     → 목표, 메트릭, 범위, 에이전트 체인  │
│                                  │
│  2. 에이전트 체이닝 실행            │
│     → agent A → handoff → agent B → ...  │
│                                  │
│  3. 사이클당 평가                  │
│     → 메트릭 측정 → keep/discard   │
│     → Git 커밋 (keep) / reset (discard)  │
│     → results.tsv 기록            │
│                                  │
│  4. 시간 예산 소진까지 반복          │
│     → 다음 사이클 자동 설계          │
│                                  │
│  5. 완료 → 보고서 생성 + 메신저 전송   │
└─────────────────────────────────┘
    ↓
아침에 결과 확인
```

## 4. `program.md` 설계

```markdown
# Program: {프로젝트명}

## 목표
{무엇을 달성할 것인가}

## 메트릭
- {metric_name}: {설명} (threshold: {값})

## 에이전트 파이프라인
1. {agent_name}: {이 단계에서 할 일}
2. {agent_name}: {이 단계에서 할 일}
3. ...

## 범위
- 수정 가능: {파일/디렉토리 목록}
- 수정 불가: {파일/디렉토리 목록}

## 시간 예산
{N}시간 또는 {N}사이클

## 종료 조건
- {메트릭} > {threshold} 달성 시
- 시간 예산 소진 시
- {N}회 연속 discard 시 (교착 탈출)
```

## 5. 실험 추적

**results.tsv** (append-only):
```tsv
cycle	timestamp	agent	metric	value	status	commit	description
1	2026-03-19T22:05	desk-researcher	signal_relevance	0.45	discard	-	초기 시장 스캔
2	2026-03-19T22:32	desk-researcher	signal_relevance	0.72	keep	a1b2c3d	타겟 커뮤니티 집중 스캔
3	2026-03-19T23:01	data-analyst	insight_actionability	0.68	keep	d4e5f6g	패턴 분석
```

**Git 커밋 컨벤션:**
```
[cycle-{N}] {status} {agent}: {description} ({metric}={value})
```

## 6. CLI

```bash
# 밤새 자율 실행
solo-agents run --program program.md --hours 8

# 단일 사이클만 실행 (디버깅)
solo-agents run --program program.md --cycles 1

# 현재 진행 상황 확인
solo-agents run --status
```

## 7. 구현 체크리스트

| # | 작업 | 변경 대상 | 설명 |
|---|------|----------|------|
| 1 | program.md 템플릿 | `assets/templates/program.md` | 자율 실행 방향 정의 |
| 2 | results.tsv 템플릿 | `assets/templates/results.tsv` | 실험 로그 포맷 |
| 3 | program.md 파서 | `src/engine/program-parser.ts` (신규) | 목표, 메트릭, 파이프라인, 종료 조건 파싱 |
| 4 | 에이전트 체이너 | `src/engine/agent-chainer.ts` (신규) | 파이프라인 순서대로 에이전트 실행 + handoff 자동 전달 |
| 5 | keep/discard 판단 | `src/engine/evaluator.ts` (신규) | 메트릭 기반 자동 판단 + Git 커밋/리셋 |
| 6 | 실험 트래커 | `src/engine/tracker.ts` (신규) | results.tsv 관리 + 사이클 로깅 |
| 7 | `solo-agents run` CLI | `src/cli/run.ts` (신규) | 자율 실행 진입점 |
| 8 | 루틴 학습 연동 | `src/scheduler/routines.ts` | 기존 루틴에 keep/discard 메트릭 추가 |
