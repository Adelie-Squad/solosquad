# v1.2.1 — Slack 연결 실패 디버깅

> 다른 데스크탑에서 `solosquad init`으로 Slack 설정을 완료해도 `solosquad bot` 실행 시 Slack이 연결되지 않는 이슈를 추적/수정한다.

**기준 버전:** v1.1.2 (npm 퍼블리시)
**영향 범위:** Slack / Discord / Telegram 전 플랫폼 (Slack에서 가장 뚜렷하게 발현)
**심각도:** Critical — 초기 사용자가 첫 실행에서 반드시 부딪히는 경로

---

## 1. 재현 경로 (사용자 리포트)

```
1. 타 데스크탑에서 npm install -g solosquad
2. solosquad init → Slack 선택 → xoxb-..., xapp-... 입력
3. "✓ .env saved", "Setup Complete!" 메시지 확인
4. Slack 앱에 봇 생성 후 워크스페이스 설치 완료
5. solosquad bot 실행 → Slack 연결 안 됨
```

---

## 2. 근본 원인 (Root Cause)

### 2.1 치명적: `.env`가 `process.env`로 로드되지 않음

| 항목 | 상태 |
|------|------|
| `package.json` dependencies | `dotenv ^16.4.0` 선언됨 |
| `dotenv.config()` 호출 | **없음** (전체 소스 grep 결과 0건) |
| `import "dotenv/config"` | **없음** |
| 영향 | `.env`에 저장된 모든 값이 런타임에서 보이지 않음 |

**호출 체인:**

```
bin/solosquad.ts
  → src/cli/index.ts          ← dotenv 로드 없음
  → src/bot/index.ts::startBot()
  → src/messenger/index.ts::createAdapters()
      → process.env.MESSENGER  ← undefined, "discord"로 폴백
  → SlackAdapter.startBot()
      → process.env.SLACK_BOT_TOKEN  ← undefined
      → process.exit(1)
```

### 2.2 증상이 사용자에게 명확하지 않은 이유

`MESSENGER` 환경변수가 `process.env`에 없으면 `createAdapters()`는 Discord로 폴백한다 (`src/messenger/index.ts:28, 35`). 따라서 사용자가 Slack을 선택했더라도:

- **경로 A:** 기본적으로 Discord 어댑터가 시작되고, Discord 토큰도 없어 `process.exit(1)`. Slack은 아예 시도되지 않음.
- **경로 B:** 사용자가 쉘에서 `export MESSENGER=slack`을 따로 한 경우에만 Slack 어댑터가 로드되고 "SLACK_BOT_TOKEN and SLACK_APP_TOKEN are required"가 출력됨.

→ 즉 사용자의 "Slack 연결이 안 됨"은 사실 **Slack 어댑터가 로드조차 되지 않은 상태**일 가능성이 크다.

## 3. 부차적 가능 원인 (2차 조사 대상)

Root cause 해결 후에도 Slack이 연결되지 않을 때 점검해야 할 항목.

### 3.1 Slack 앱 설정 이슈

| 항목 | 확인 방법 | 필수 여부 |
|------|----------|----------|
| Socket Mode 활성화 | api.slack.com/apps → 앱 → Socket Mode → Enable | 필수 (어댑터가 `socketMode: true`) |
| App-Level Token (`xapp-`) scope | `connections:write` 포함 | 필수 |
| Bot Token (`xoxb-`) scopes | `channels:read`, `channels:manage`, `chat:write`, `groups:read`, `app_mentions:read`, `channels:history` | 필수 |
| Event Subscriptions | `message.channels` 이벤트 구독 | 필수 |
| 워크스페이스 재설치 | scope 변경 후 Reinstall App 필요 | 조건부 |
| `#owner-command` 채널 봇 초대 | 채널에서 `/invite @bot` | 필수 |

`slack-adapter.ts:71`의 `channelName !== COMMAND_CHANNEL` 체크로 인해, 봇이 해당 채널에 없거나 다른 채널에서 메시지를 받으면 **조용히 무시**된다. 이것도 "연결 안 됨"으로 오인될 수 있음.

### 3.2 토큰 포맷 오류

| 항목 | 정상 포맷 | 흔한 실수 |
|------|----------|----------|
| Bot Token | `xoxb-<숫자>-<숫자>-<문자열>` | `xoxp-`(User Token) 입력 |
| App Token | `xapp-1-<문자열>-<숫자>-<문자열>` | Signing Secret 입력, 만료된 토큰 |
| `.env` 공백/따옴표 | `SLACK_BOT_TOKEN=xoxb-...` | 따옴표 포함 시 `saveEnv`가 그대로 보존 |

### 3.3 네트워크

Socket Mode는 `wss://wss-primary.slack.com` 아웃바운드 WebSocket이 필수. 회사 방화벽/VPN에서 차단될 수 있음.

## 4. 수정 계획

### Phase 1 — Root cause 수정 (필수)

| 단계 | 변경 | 파일 |
|------|------|------|
| 1-A | `bin/solosquad.ts` 최상단에 `import "dotenv/config"` 추가 | `bin/solosquad.ts` |
| 1-B | `saveEnv` 후 `process.env`에도 즉시 반영 (init 이후 동일 프로세스에서 다른 명령이 실행되는 경로 대비) | `src/util/config.ts` |
| 1-C | `doctor`가 `process.env` 가 아니라 `loadEnv()` 결과를 검사하고 있어 버그 은폐 중 — `doctor`에서 `process.env` vs `.env` 불일치를 경고로 표시 | `src/cli/doctor.ts` |

### Phase 2 — 사용자 진단 가능성 향상

| 단계 | 변경 | 파일 |
|------|------|------|
| 2-A | 어댑터가 `process.exit(1)` 전에 **플랫폼별 원인**과 **다음 조치**를 출력 (예: "xoxb/xapp 포맷 확인, Socket Mode 활성화 확인") | `src/messenger/slack-adapter.ts`, 기타 어댑터 |
| 2-B | `bot` 커맨드 시작 시 현재 로드된 `MESSENGER` 값을 출력 — 현재는 `[Bot] Starting with adapters: ...`만 나옴. 실제 `process.env.MESSENGER`가 어디서 읽혔는지 (process env / .env / default) 명시 | `src/bot/index.ts` |
| 2-C | Slack 어댑터에 `app.error()` 핸들러 추가하여 Bolt 내부 에러(스코프 부족, Socket 연결 실패 등)를 콘솔에 노출 | `src/messenger/slack-adapter.ts` |
| 2-D | `doctor`에 `--messenger-check` 옵션 추가: 토큰으로 Slack `auth.test` API 호출하여 실제 유효성 검증 | `src/cli/doctor.ts` |

### Phase 3 — 문서/회귀 방지

| 단계 | 변경 | 파일 |
|------|------|------|
| 3-A | `setup-guide.md`에 Slack 앱 설정 체크리스트 추가 (Socket Mode, scopes, 봇 채널 초대) | `docs/setup-guide.md` |
| 3-B | README Slack 섹션에 "Event Subscriptions + `message.channels`" 명시 | `README.md` |
| 3-C | `test/` 에 `.env` 로드 확인 스모크 테스트 (임시 `.env` 생성 → `solosquad doctor` 실행 → 토큰 감지 확인) | `test/env-load.test.ts` (신규) |

## 5. 검증 체크리스트

Phase 1 수정 후 다음이 모두 통과해야 한다.

- [ ] 새 셸에서 `export MESSENGER=` 하지 않고 `solosquad bot` 실행 → `.env`의 `MESSENGER=slack` 값이 로드되어 Slack 어댑터가 시작됨
- [ ] `solosquad doctor` 가 `.env`의 값과 `process.env` 값 둘 다 참조하여 일관된 결과 출력
- [ ] 잘못된 토큰 (xoxp-) 입력 시 Phase 2-A 메시지 출력
- [ ] Slack 앱 설정 누락 시 (Socket Mode off) Bolt 에러가 콘솔에 노출됨
- [ ] 타 데스크탑 재현: `init` → `bot` 2-커맨드만으로 `[Slack Bot] Starting Socket Mode...` 로그 출력
