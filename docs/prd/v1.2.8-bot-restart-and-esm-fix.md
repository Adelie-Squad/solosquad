## 1. 발견 경위

`solosquad@1.2.7` publish 직후 dogfood (2026-05-29 KST 오후) 에서 다음 시퀀스:

1. CLI 1.2.7 + workspace migrate → **외관상 정상**
2. Discord 에서 "bv-po-homepage-nextjs README 읽어줘"
3. Chief: *"권한 승인 프롬프트 — `/add-dir C:\Dev\... 직접 입력해 주세요"*
4. v1.2.7 의 `--add-dir` 자동화가 *논리적으론 완벽한데도* 실제 spawn 에 안 들어감

진단 4단계 (4 모두 PASS):
- (A) `claude --print --add-dir <path>` 단독 호출 → ✓ README 정상 read
- (B) `<org>/repositories/*.yaml` 파싱 → ✓ 3 path 모두 있음
- (C) installed `dist/src/bot/chief-runner.js` grep → ✓ wiring 코드 박제
- (D) 같은 코드 ESM context 에서 실행 → ❌ `ReferenceError: require is not defined`

→ **`chief-runner.ts` 의 helper 들이 `require("fs"|"path"|"js-yaml")` 사용 → package 가 `"type": "module"` → ESM 컨텍스트에서 `require` undefined → throw → 바깥 `try/catch` 가 silently swallow → `addDirs` 빈 배열 → `--add-dir` spawn 에 안 들어감.**

추가 발견 항목 (같은 fix 직전에 발견):
- **봇 재기동 UX 부재** — `migrate --apply` 후 봇 자체는 옛 1.2.6 코드를 메모리에 들고 있음. 사용자가 매번 Ctrl+C + 재기동해야 새 코드 반영. *클라우드든 로컬이든 귀찮음*.
- **작업 중 봇 종료 위험** — Chief turn 진행 중 SIGTERM 받으면 즉시 종료. 응답 손실 + claude 자식 프로세스 orphan + 사용자 침묵.

이 3건 (ESM fix + 재기동 wiring + drain) 모두 단일 publish 슬롯에 통합.

## 3. v1.2.x 흐름 정합 — 권한 모델 매트릭스

v1.2.x 의 3 layer 권한 인프라가 v1.2.8 에 와서 완성:

| Layer | 무엇을 함 | 도입 버전 |
|---|---|---|
| **Trust dialog 박제** (`~/.claude.json projects[<path>].hasTrustDialogAccepted=true`) | 디렉토리 자체에서 작업 시작 가능 — Claude 가 진입 거부 안 함 | v1.2.6 §A.5 |
| **Cross-cwd file access** (`claude --add-dir <path1> <path2> ...`) | cwd 밖 path 도 Read/Edit/Write 가능 | v1.2.7 §A.6 *(wiring 만 완성, 실작동은 v1.2.8 §A.9)* |
| **Bot 재기동 자동화** (PID file + migration SIGTERM + `--supervise`) | 사용자가 Ctrl+C / 수동 재기동 안 해도 새 코드 반영 | v1.2.8 §A.10 |

세 layer 합쳐서 = 사용자가 *어떤 권한 다이얼로그도 안 보고 / 어떤 봇 명령도 안 입력하고* 모든 등록된 repo 에 봇이 작업 가능.

## 4. v1.2.7 의 *모든 게이트 통과* 원인 분석

`tsc --noEmit`, 728 test, `docs-check` 모두 PASS 한 채 broken 출시된 이유:

| 게이트 | 왜 안 잡았나 |
|---|---|
| `tsc --noEmit` | `require("fs") as typeof import("fs")` 의 `as` 캐스트가 TypeScript 에 *적법한 result 타입* 보여줌 → ESM/CJS mismatch 무시. 컴파일 결과 `.js` 에서는 그대로 `require("fs")` 출력 |
| 728 test | `collectRegisteredRepoPaths` 가 unit-test 안 됨 — 함수가 *bot spawn 시 runtime injection* 으로만 호출됨. tsx 가 test 를 CJS 호환 모드로 transpile 해서 *어쩌면 통과시켰을* 가능성 (확실 안 함) |
| `docs-check` | 문자열 매칭만. 코드 동작 영역 밖 |
| Manual 재현 | publish 후에야 사용자 dogfood 로 발견 |

**v1.2.8 의 esm-check (§A.11)** 가 이 gap 의 *마지막 자물쇠* — compiled `.js` 안의 `require(` 를 1초 안에 catch.
