## 2. 스코프 (In)

### 2.1 내장 `research` 워크플로우 (workflow-manager 자산)
- 신설 `skills/workflow-manager/assets/workflows/research/` — 기존 워크플로우 구조(`program`/stages) 준수.
- **메인 워크플로우로 등재**(현 2 메인 new-build·improvement → **3번째 research**). Chief 맥락 추론으로 선택.
- 단계(이번 dogfood 일반화): `spec(주제·rubric 고정) → 확산→수렴 가설수립 → 실험 사이클(plan→run→result) → 백테스트/수치 eval → Track2 verifier 검증 → 리포트/논문 산출`. 사이클은 수렴(rubric 통과) or 예산까지 반복.
- 기존 자산 재사용: `hypothesis-design`·`experiment-design`·`market-research`·`discovery-synthesis` 스킬을 파이프라인 단계로 조립(발명 아님).

### 2.2 research 템플릿 + 설계 예시 내장
- 현 `docs/research/research-subject/`(주제 템플릿: research-plan·26xxxx_cN_experiment(plan/result)·final-result)를 **워크플로우 자산으로 이관**(`.../workflows/research/assets/subject-template/`). **`docs/research/` 에는 실제 실험 주제 dir만 남긴다.**
- **기본 설계 예시 임베드**: (a) 연구계획 예시 (b) research-workflow(확산→수렴·수치 eval) 규약 (c) **백테스트 하네스 예시**(blind·누출통제·baseline·Δ/LOO). 이번 시뮬 연구를 축약한 golden example.

### 2.3 goal 무인 8h+ 장기실행
- `skills/goal-manager/SKILL.md` + 장기실행 지침에 반영(엔진 `src/engine/**` 불변 — 스킬·스캐폴드·시스템프롬프트만):
  - **확산→수렴 가설수립**, **증거앵커 진행보고**(no fabrication), **자율 파이프라인 reminder**(무인·되돌릴수있으면 진행), **체크포인트 정지**(파괴/비가역만), **예산 인식**(카운트다운 노출 억제·도달 시 요약), **verifier 서브에이전트 > 자기비평**.
  - **최소 기준: 사용자 지시 없이 8h+ 지속** (Ralphthon 3h 무인을 넘어). budget·idle 안전경계 4조건 AND continuation.
- 근거: `docs/ideation/260712-long-horizon-codex-goals-vs-fable5.md`(Codex goal 6필드 + Fable 프롬프트).

### 2.4 리포트 위치 이전 (`docs/reports` → org `reports`)
- **문제:** `docs`는 `package.json.files`로 배포 안 됨 → 사용자가 최종 산출물(리포트·PPT·논문·프로토타입)을 못 받음. reports는 **org 단위 배포 산출물 공간**이어야 함.
- **변경:** `market-research` 스킬 산출 경로 `<org>/docs/reports/` → **`<org>/reports/`**. `docs` 스킬·v1.3.5/v1.3.8 참조·architecture 갱신(fix-forward: frozen PRD의 `evidence_ref`는 안 옮김).
- **마이그레이션 `1.4.2-to-1.4.3`:** 기존 `<org>/docs/reports/` → `<org>/reports/` 이동(존재 시), `docs/reports/INDEX.md` 제거. 데이터 보존(class A).
