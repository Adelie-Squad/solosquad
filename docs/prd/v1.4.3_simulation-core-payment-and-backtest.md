## 1. Summary

세 조각을 최소 버전으로 세운다: ⑴ **행동 3분류 시뮬레이션 코어**(관심·**전환(결제)**·이탈), 특히 페르소나가 결제를 **확률적으로 결정**하는 Fake-door 엔진, ⑵ **사례 재현 백테스트 하네스**(Dropbox·Zappos·New Coke·IBM·Buffer 익명화 → 예측 vs 실제 비교), ⑶ **4h+ 장기 research 하네스**(codex goal + Fable 로 가설→실험→검증 반복, 산출 `/reports`). AI 제품 기능이므로 **Eval Plan(백테스트)이 곧 효과성 지표**.

## 2. Background (우산 §1·§4·§5 압축)

- 수요검증(인터뷰·Fake door·A/B)은 권장되나 자원 제약 → 합성 페르소나 시도, 그러나 **실세계 유효성 미검증**(우산 §1).
- 본 patch 는 그 유효성을 **백테스트로 정면 측정** — 5개 실제 사례의 결과를 시뮬레이션이 재현하는지.

## 3. 핵심 개념

### 3.1 행동 3분류 코어
```
Persona(속성) × Stimulus(제품/랜딩/가격) → Decision
  관심 행동:  view / like / click / email-submit      (관심 지표)
  전환 행동:  purchase (확률적 결정)                    ← 핵심
  이탈 행동:  bounce / churn / refund                  (이탈 지표)
```
- **결제(전환) 결정:** 페르소나 속성(니즈·지불의사·가격민감도 등) → 결제 확률 `p_pay` 유도 → 확률적 샘플링. (사전분포·캘리브레이션 = §Open Q)

### 3.2 수요검증 방법론 (v0 = Fake door 우선)
- **Fake door(필수):** 각 페르소나가 클릭/이메일/결제를 확률적으로 결정 → 전환 퍼널 집계.
- **A/B(최소):** 두 변수(가격/카피 등) 병렬 노출 → 전환율 승자.
- **인터뷰(스텁):** 특정 페르소나 대화형 심층 질의 — v0 은 인터페이스만, 심화는 후속.

### 3.3 백테스트 하네스
- 5사례를 `{배경, 실제 검증방법, 실제 결과}` 로 익명화 정형화 → 동일 조건 시뮬레이션 → **예측 방향·강도 vs 실제** 판정 → `/reports` 리포트.

### 3.4 장기 research 하네스 (4h+)
- codex goal cycle + Fable 장기 세션으로 **가설 수립 → 실험 실행 → 검증 → 다음 가설** 자율 반복. 각 사이클 레코드를 `/reports` 에 append(감사·재현 가능).

## 4. 접근 옵션 (≥2)

### A. 결정적 시뮬레이터 코어 + LLM 은 페르소나 판단에만 (권고)
행동 퍼널·집계·백테스트 비교는 **결정적 코드**, 개별 페르소나의 결제/이탈 "판단"만 LLM(Fable) 호출.
- **장점:** 재현성(백테스트 신뢰)·비용 통제·효과성 지표가 코드에 고정. engine 경계(`src/engine/**` 참조만) 준수.
- **단점:** 페르소나 판단 호출 수 × 세그먼트 크기 → 토큰/시간 관리 필요(4h+ 하네스가 흡수).

### B. 전면 에이전트 시뮬레이션 (각 페르소나 = 자율 에이전트)
모든 행동을 에이전트 자율 상호작용으로.
- **장점:** 창발적 현실성.
- **단점:** 재현성·백테스트 판정 난해, 비용 폭증, 효과성 "입증"이 어려움. → **v0 기각**(후속 실험 트랙).

**선택 = A.** 효과성 *입증*이 본 우산의 목적이라 재현 가능한 결정적 골격이 먼저. 페르소나 현실성은 B 요소를 후속 patch 로 점진 도입.

## 5. 스코프 (v0)

**In:**
