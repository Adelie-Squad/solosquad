## 1. 무엇을 만드나 (한 줄)

`research`를 **내장 메인 워크플로우**로 추가하고(기존 workflow-manager가 관리), 그 워크플로우가 **확산→수렴 가설수립 → 실험 사이클 → 백테스트/수치 eval → Track2 검증 → 논문/리포트 산출**을 자율로 돌리게 한다. 장기 실행(무인 8h+)은 goal 프로세스로, 산출물은 org-level `reports/`로.

## 2. 배경 — 지금 없는 것 (코드 근거)

- **내장 워크플로우:** 현재 12종(new-build·improvement 메인 + hypothesis·market-research·data-analysis 등 서브, `skills/workflow-manager/assets/workflows/`). **"연구(가설→실험→검증 반복)" 워크플로우는 없다.** hypothesis/experiment-design 스킬은 있으나 *사이클 반복·백테스트·수치 eval·논문화*를 묶는 워크플로우가 부재.
- **goal 장기실행:** `skills/goal-manager/SKILL.md` + `src/engine/goal-runner.ts`(불변)는 metric keep/discard 루프. **무인 4h+ 장기 research 루프**의 규율(확산→수렴·증거앵커·no p-hacking·자기교정)이 명문화 안 됨.
- **리포트 위치:** `market-research` 산출이 `<org>/docs/reports/`(v1.3.5). **docs는 배포 안 됨** → 최종 산출물(리포트·PPT·논문·프로토타입)은 **org-level `<org>/reports/`**(배포 대상)에 있어야 함.
- **research 기록 체계:** 이번에 임시로 `docs/research/`에 만든 템플릿·사이클 구조가 내장 자산이 아님.

## 4. 연구 워크플로우 = 이번 dogfood의 일반화 (설계 요지)

`skills/workflow-manager/assets/workflows/research/research-workflow.md`(이번에 작성)를 내장 규약으로 승격. 핵심:
- **확산→수렴 가설수립**(§4.1): 직전 사이클 진단 입력 → PM/오케스트레이터가 **웹·문헌 조사로 후보 가설 확산** → 채점으로 하나 수렴. (내부지식만 의존 금지)
- **실험 사이클**: `plan(goal 6필드) → 실행 → result(증거앵커)` , 사이클 dir `26xxxx_cN_*`.
- **검증 = 수치 eval + Track2 verifier**: 사전등록 임계(Δ·dir-acc·LOO·ECE) + 별도 리뷰 에이전트(soundness/originality…) 채점, overall≥4 수렴.
- **산출**: 과정=`docs/research/<subject>/`, 최종=**`reports/`**(논문·리포트).
- **정직성 규율**: no p-hacking · 증거 없는 주장 금지 · 자기교정(예: C6 5/5 → C7 0.64).
