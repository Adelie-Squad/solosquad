## 4. 해결책

### 4.1 즉시 (v1.2.3 hotfix)

`package.json.files` 화이트리스트에 6 개 root dir 추가:

```diff
   "files": [
     "dist/",
     "assets/",
+    "agents/",
+    "skills/",
+    "teams/",
+    "schedules/",
+    "user/",
+    "knowledge/",
     "manual/",
     "AGENTS.md",
     "CHANGELOG.md"
   ],
```

결과: tarball 파일 수 **567 → 652** (875 kB → 935 kB). `npm pack --dry-run` 으로 chief SKILL.md, 4 teams OKR/KNOWLEDGE/composition, problem-definition workflow.yaml, 25 specialist SKILL.md 모두 listing 확인.

### 4.2 Migration 경로 정합

`src/migrations/scripts/1.1.0-to-1.2.2.ts` → `1.1.0-to-1.2.3.ts` rename, `TARGET = "1.2.2"` → `"1.2.3"`. 사용자 워크스페이스가 v1.2.2 라벨에 *멈춰 있을 수 없는* 구조 (v1.2.2 verify 가 100% 실패하므로 어떤 워크스페이스도 1.2.2 까지 도달 못 함) → forward-rename 안전.

### 4.3 사용자 복구 절차

```
solosquad migrate --rollback             # picker → 원래 v0.9.2 backup 선택
npm install -g solosquad@latest          # 1.2.3 pull
solosquad migrate --apply                # 전체 chain 재실행 (bundle 정상)
```

`1.0.4 → 1.1.0` 의 `apply()` 가 *silently skip 한* seeds 를 재실행하기 위해 `1.0.4` 까지 되돌리는 것이 필수. v0.9.2 backup 이 가장 깊고 안전.

## 6. 회고 — 왜 publish-gate 가 못 잡았나

`npm run prepublishOnly` 는 `tsc + docs-check` 만 실행. tarball 내용물 검증 없음.

**관련 사실들:**
- Tests 는 워크스페이스의 *on-disk* bundle 을 읽음 (intact) → npm-install-then-migrate 경로를 simulate 안 함
- `npm pack --dry-run` 출력은 file count + size 표시. 특정 seed 경로 manifest 와 cross-check 안 함
- v1.1.0 publish 도 같은 결함 (1.1.0 → 1.1.0 자가 migration 이 없어서 사용자가 v1.0.x → v1.1.0 chain 안 돌리면 모름 — sleeping bug 였음)

**v1.2.4 follow-up 후보 (별도 plan)**:
1. **`prepublishOnly` 에서 tarball seed-path manifest 검증** — `collectSeeds()` 반환을 기준으로 `npm pack --dry-run` 출력을 grep
2. **`migration apply()` 의 silent seed-skip → abort with attributable error** — bundle source 부재 시 silently skip 대신 *어느 파일이 어디 누락됐는지* 명시 + verify 전 실패
3. **`scaffoldOrg.copyBundleFile()` 의 silent skip** 도 동일 정합 (add-org 결함 노출)

→ 위 3건 모두 `docs/prd/v1.2.6-onboarding-and-vocabulary-polish.md` §X (silent-skip → abort 카테고리) 슬롯.
