### 0.2 rev4 로 폐기된 논거 2건 (재사용 금지)

1. **"세션 훅이 TS 를 강제한다" / "Py 의 유일한 대가는 훅 shell 우회"** — ❌ **사실오류.**
   SoloSquad 는 **Claude Agent SDK 를 쓰지 않는다.** `package.json` 에 `@anthropic-ai/*` 0건이고,
   `src/bot/claude-process.ts:1` 이 `claude --print --output-format stream-json` **서브프로세스**를 spawn하며,
   훅은 이미 `.claude/settings.json` **shell 훅**(`src/bot/dev-confirm-hook.ts:14`, PreToolUse)이다.
   → 그 비용은 **TS 현행이 이미 치르고 있으므로 Py 전환의 대가가 아니다(=0).** 동시에 *"세션훅 vs 시뮬밀착"* 이분법도 한 축이 비어 **결정 기준으로 성립하지 않는다.**

2. **"시뮬레이션·분석 밀착이 본질이므로 Py"** — ❌ **자기 리포 연구가 반증.**
   `docs/research/virtual-market-demand-simulation/final-result.md:5` 종합 판정 = *raw 시뮬 **무효(反예측)** · 캘리브레이션 부분 회복(~0.64 정체) · 완전 유효 미도달 · N=14 저검정력*.
   그에 맞춰 `docs/prd/v1.9.0_virtual-market-demand-simulation.md` 도 이미 *"원시 예측기가 아니라 스크리닝 특징 + 진단 플래그"* 로 축소됐다.
   → 시뮬은 **핵심 루프가 아니라 v1.9.0 의 보조 기능**이며 현 마일스톤에서 4단계 뒤다. **언어 결정 근거에서 제외.** 필요 시점에 Py 서브프로세스/서비스로 붙이면 되고, 그것은 지금 72k LOC 를 옮길 이유가 못 된다.

### 0.3 "Py 전면" 기각 사유

- §0.1 전환 비용이 1인 유지보수 상한을 초과. 특히 migrations 43종은 **사용자 상태 계약**이라 재작성으로 해소되지 않는다.
- 언어 경계가 줄지 않는다 — 오케스트레이션만 Py 로 가면 **봇(TS, `messenger/`+`bot/` 14.3k LOC) ↔ 오케(Py) 경계가 신설**된다. 폴리글롯은 이 경계를 IPC 로 **정직하게 인정**하는 안이고, "경계 1곳" 주장은 이 경계를 세지 않은 계산이다.
- **배포 채널 교체(npm→pipx 주채널)도 함께 기각**: 현행 전제조건은 Node 1개인데 제안안은 *Python 3.10+ 확인/설치 → pipx → Docker* 로 **비개발자에게 더 무겁다**. `curl/irm` 원클릭은 언어와 무관하게 지금도 가능. → **npm 주채널 유지**, Py 측은 내부 의존성으로 격리.
- **모노레포에 `apps/dashboard/` 편입도 불채택**: `docs/prd/v1.x-dashboard-interaction.md:3` 이 이미 *"웹 대시보드는 별도 리포(`solopreneur-dashboard`/`solopreneur-api`), 본 리포는 인터페이스만"* 으로 결정. 웹은 오케스트레이션 언어를 강제하지 않으므로(이 명제는 유효) 대시보드를 끌어들일 이유도 없다.

### 0.4 ADR 을 닫는 방법 — 문서가 아니라 스파이크

> 전면 결정을 문서로 확정하지 말고 **1개 어댑터로 실측**한 뒤 확대한다.

1. **하네스 인터페이스 먼저 정의**(spawn·session·tool·stream·budget) — 언어 중립 계약.
2. **Codex CLI 백엔드 어댑터 *하나만* Py 로 스파이크.** 동일 인터페이스의 TS 구현과 비교.
3. **측정 항목:** MCP/툴링 생태계 실이득 · IPC 오버헤드 · 1인 유지 부담(빌드·테스트·배포 파이프라인 2중화 비용).
4. 이득이 확인될 때만 오케스트레이션/하네스로 확대. 미확인 시 **TS 유지가 기본값**이다.
5. **언어 상한 2개 유지**(1인 제약) — 현행 TS 1개 + Py 1개까지. Rust 등 추가는 이 단계에서 금지.
