## ⚡ 0. Track 0 — 플랫폼 결정 (최우선 게이트, 나머지를 게이팅)

> **두 번 짓지 않으려면 먼저 확정.** 아래가 정해지기 전 Track A~D 착수 금지.

- **언어 ADR:** TS 유지 / **폴리글롯**(봇·CLI=TS, 오케스트레이션·하네스·연구 코어=Py, IPC로 연결) / Py 전면. — 스파이크 + `docs/policy/` ADR.
  - *유혹(Py):* Codex·에이전트/LLM 툴링·MCP 생태계·이번 연구 스택이 Python 중심.
  - *비용(전환):* 현 제품 전체(1026 test · npm 배포 · 봇 · 마이그레이션)가 TS. → **권고: 폴리글롯**(봇/배포 TS 보존 + 오케스트레이션/하네스는 Py).
- **백엔드 추상화:** Claude Code ↔ **Codex CLI** 를 **공통 하네스 인터페이스**(spawn · session · tool · stream · budget)로. (구 `v1.x-llm-backend-abstraction.md` 승격.)
- **산출:** 언어 ADR + 하네스 인터페이스 정의. **HARD GATE.**

## 1. 트랙 (Track 0 확정 후)

### Track A — 오케스트레이션 코어 (구 세션 Track A 일반화)
명시적 primitive 로: **memory** · **router**(입력→세션/백엔드 라우팅, bot `classifyIncoming` 확장) · **context** · **handoff** · **cwd**(=S-6 실행컨텍스트 리졸버) · **다중 세션/디스코드 스레드 구분**(=S-4 per-과제 워커) · **역할분리**(=S-5 오케스트레이터↔워커) · **토큰 임계 회전**(=S-2b).
- **HARD GATE:** goal/workflow 스폰 진입점 스파이크 (engine `src/engine/**` 참조만).

### Track B — 토폴로지 프레임워크
에이전트/세션 위상을 **명시적 데이터 모델**로: 어느 에이전트가 어느 세션·cwd·백엔드에서 돌고 누구에게 handoff 하는가. (S-5 + v1.1 팀 아키텍처 일반화.) Track A 의 정적 스키마.

### Track C — 하네스 엔지니어링
**hook**(실행 전/후 개입점) · **가드레일**(파괴/비가역/스코프 경계 — v1.3.0 dev-confirm 게이트의 일반화). **백엔드 무관 공통 층** — Claude Code hook + Codex 동형.

### Track D — 도구 · 시크릿 · 통합
**MCP · API · 토큰 · key · 로그인/인증 · CI/CD 디렉토리** 관리 + **워크플로 반영**(도구를 워크플로 stage 가 쓰게). 시크릿 마스킹(v0.2) · MCP(구 `v1.x-knowledge-ontology`) 확장.

### Track E — 커스텀 Reconciliation (구 M1~M4, 우선순위↓)
overlay tier(M1) · seed manifest + `update --reconcile`(M2) · health-gated rollback(M3) · 플러그인 정렬(M4, PoC). **격변과 독립·병행 가능** → 후순위.

### Track F — 잔여 세션 조각
cron 자산 통합(**S-7**) · GC(**S-3b** — `archive-rotate` 이미 라이브라 재정의 필요).
