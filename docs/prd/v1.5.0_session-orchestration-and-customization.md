### A.3 S-2b — 토큰 임계 세션 회전 (구 §5.1)

세션 토큰이 임계 도달 시 핸드오프 요약 후 세션을 갈아끼운다. **S-2a(관측)가 이미 토대**를 깔아둠(`chief.usage` 이벤트, `contextTokens` 캡처).

- **설정:** `ChiefConfig.compaction?: { warn_pct?; rotate_pct?; enabled? }` (yaml `chief.compaction`, 기본 **0.70 / 0.85 / true**). 현행 `config.ts` 는 `compaction_time` 만 보유 → 신설.
- **분모:** pct = `contextTokens / 컨텍스트 윈도우`. 윈도우 상수/모델별 매핑 신설(없으면 char-heuristic fallback, 리스크 #1).
- **흐름:** `warn_pct` → Chief 에 핸드오프 요약(`memory/pm-skills/<wf-id>.md`) 지시 → `rotate_pct` → **md 작성 성공 확인 후** `session-store.rotate(…,"auto-compaction")` → 새 세션 첫 입력에 요약 주입 → `chief.session_rotated` 이벤트.
- **재사용:** `session-store.rotate` + `archived[]`(이미 존재).
- **리스크(구 §8):** ⑤이중 압축(Claude Code 네이티브 auto-compact 와 겹침) — 명시적 임계+가시 이벤트로만. ⑥회전 중 유실/재회전 루프 — usage 카운터 리셋 명시, 크래시 시 `archived[]` 조회.
- **게이트:** 사이드이펙트 위험군 A(멀티턴 컨텍스트 증발·크래시-후-미전달 유실·회계 리셋 루프) 단일세션 검증 후 이행.

### A.5 S-4 — per-repo/과제 워커 세션 (구 v1.4.4, 간판)

봇이 과제(works 스레드) 단위로 **별도 claude 프로세스를 스폰**. 자식 세션 키 `(userId, orgSlug, taskId)` → `sessions/<user>__<task>.json`, cwd = 과제 scope repo(S-6), 수명 = 스폰→hydrate→flush→kill(요약만 메인 Chief 환류). v1.4.1 Approach A(공유 세션)를 **격리 자식 세션(Approach B)** 으로 승격. §5.6 goal 중간개입도 같은 가로채기 인프라로.

### A.6 S-3b — GC (⚠️ 코드 실측으로 재정의 필요)

## Track B — 커스터마이즈 ↔ Upstream Reconciliation (구 v1.5.0 병합)

> 원 설계는 삭제된 구 `v1.5.0-customization-upstream-reconciliation.md`(git 히스토리 보존). 아래는 요지 병합; 설계 원칙(보존 회귀 0·편집이 아니라 오버레이·불투명 강요 금지·런타임 정렬 우선)은 그대로 승계.

### B.1 M1 — User-global overlay tier 실배선 (저위험·즉효)

`paths.ts` 의 `getAgentsDir`/`getSkillsDir`/`getRoutinesDir`/`getTeamsDir` 등 해석을 **3-tier 확정**: `워크스페이스 .solosquad/<x>/` → `~/.solosquad/<x>/`(신규 배선) → 번들. 현재 `~/.solosquad/` 는 `agent-profile-defaults.yaml` 하나만 읽혀 광고된 override 존이 inert. oh-my-zsh `custom/` 동형 "절대 안 덮는 존" 완성.

### B.2 M2 — Seed manifest + `update --reconcile` (copier 식 3-way)

- `init`/`migrate` 가 `<ws>/.solosquad/.seed-manifest.yaml` 에 `{path, engine_version, sha256}` 박제(copier `_commit` 등가).
- `solosquad update --reconcile`: 옛 기본값 재현 → 사용자 diff 추출 → 새 기본값에 재적용(충돌 시 git 마커/`.rej`) → 안 건드린 파일(sha 일치)은 조용히 교체.
- escape hatch: `--recopy`(새 기본값 재생성 + `.bak` 백업).

### B.3 M3 — Health-gated auto-rollback (OpenClaw 식)

`migrate --apply` 파이프라인을 **백업 → 적용 → `doctor` 헬스체크 → 실패 시 직전 백업 자동 restore + 봇 재기동** 으로 확장. v1.2.8 graceful-drain 과 결합. reconciliation 과 직교(저비용·즉효) → M1 과 선출시 후보.

### B.4 M4 — Claude Code 플러그인 정렬 (실험·장기)

기본 skill/agent 를 Claude Code 플러그인으로 패키징 → `.claude/` shadow-override(project>user>plugin) 로 커스텀 + 버전 bump 로 전파. **본 우산에선 PoC/조사만**, 본격 채택은 후속 메이저.
