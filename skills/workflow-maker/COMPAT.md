# `workflow-maker/` — frozen back-compat artifact (do not add a SKILL.md here)

The `workflow-maker` skill was renamed to **`workflow-manager`** in v1.3.5
(PRD `docs/prd/v1.3.5-planning-workflows.md` §3.9 B-D1). The canonical skill,
all templates, and every live reference now live under `skills/workflow-manager/`.

This directory survives **only** because the published, immutable migration
`src/migrations/scripts/1.1.0-to-1.2.6.ts` hard-codes a package-integrity guard:

```
verify(): getBundleRoot()/skills/workflow-maker/assets/workflows/problem-definition/workflow.yaml must exist
```

Published migrations are immutable contracts (AGENTS.md "Immutable paths").
Renaming the bundle dir away would make that migration's `verify()` fail for
anyone upgrading through v1.2.6, so we keep a frozen copy of the single file it
asserts on. It intentionally has **no `SKILL.md`** — the skill loader / scanners
skip dirs without one, so this does not register as a second skill.

Do not edit this file's `workflow.yaml` to track template changes; the live
template is `skills/workflow-manager/assets/workflows/problem-definition/workflow.yaml`.
